# System Strategy

这份文档把当前项目的核心方向固定下来，避免后续开发在地图、数据源和合规策略上来回摇摆。

## 1. 地图策略

- 主使用前端：高德地图
- 内部导出与演示：Google Earth / KML

### 为什么这样定

1. 中国本地浏览、筛选、点击下钻、GeoJSON 覆盖物和弹窗交互，更适合高德地图。
2. Google Earth 更适合研究、演示、飞行巡检和结果分享。
3. 不把中国本地研究前端压在 Google 地图栈上，可以减少后面维护的不确定性。

## 2. 坐标策略

- 原始坐标单独保留
- 中国 Web 地图展示优先使用 GCJ-02
- Google Earth 导出时统一产出 WGS-84

## 3. 数据获取策略

优先级从高到低：

1. 公开开放数据
2. 浏览器公开页抓取批次
3. 高德等个人开发 Key 支持的地图 / 几何辅助能力
4. 复核只处理抓取缺口，不再开放人工录入入口

## 4. 地址标准化模型

统一按以下层级建模：

`district -> resblock -> building -> unit -> floor`

这会直接影响：

- 去重质量
- 小区归一
- 楼栋聚合
- 楼层回报率分析
- 地图弹窗与导出精度

## 5. MVP 到自用稳定版的路线

### MVP

- 区级热度
- 小区级租售比
- 楼栋聚合示例
- KML / GeoJSON 导出

### Beta

- 真实小区字典
- 地址解析队列
- 浏览器抓取批次与复核
- 楼栋几何资产
- 指标快照任务

### Stable local

- 开放数据源接入
- AOI / footprint 补全
- 真实楼层样本明细
- 任务调度、导入审计和本地运行监控

## 6. 方向护栏

- 项目默认面向个人本地研究，不做商业化包装、销售线索、客户数据或多租户能力。
- `/` 是主使用面；`/backstage` 只服务数据质量、批次复核和本地刷新，不再扩成运营后台。
- 数据入口只保留公开开放数据、浏览器公开页抓取批次和个人地图 Key 辅助能力。
- 不新增登录、账号、计费、权限审计、外部数据平台拉取或自动化接入。
- 新功能优先级：数据可信度 > 前台日常决策体验 > 后台便利性。
