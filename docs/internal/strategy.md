# System Strategy

这份文档把当前项目的核心方向固定下来，避免后续开发在地图、数据源和合规策略上来回摇摆。

## 1. 地图策略

- 主产品前端：高德地图
- 内部导出与演示：Google Earth / KML

### 为什么这样定

1. 中国本地浏览、筛选、点击下钻、GeoJSON 覆盖物和弹窗交互，更适合高德地图。
2. Google Earth 更适合研究、演示、飞行巡检和结果分享。
3. 不把中国业务主前端压在 Google 地图栈上，可以减少后面产品化的不确定性。

## 2. 坐标策略

- 原始坐标单独保留
- 中国 Web 地图展示优先使用 GCJ-02
- Google Earth 导出时统一产出 WGS-84

## 3. 数据获取策略

优先级从高到低：

1. 官方合作接口 / 开放平台
2. 公开开放数据
3. 授权 CSV 批次导入与人工校正
4. 不稳定抓取仅用于低频研究验证，不作为长期生产主链路

## 4. 地址标准化模型

统一按以下层级建模：

`district -> resblock -> building -> unit -> floor`

这会直接影响：

- 去重质量
- 小区归一
- 楼栋聚合
- 楼层回报率分析
- 地图弹窗与导出精度

## 5. MVP 到生产的路线

### MVP

- 区级热度
- 小区级租售比
- 楼栋聚合示例
- KML / GeoJSON 导出

### Beta

- 真实小区字典
- 地址解析队列
- 授权导入批次与人工复核
- 楼栋几何资产
- 指标快照任务

### Production

- 官方数据源接入
- AOI / footprint 补全
- 真实楼层样本明细
- 权限、审计、任务调度和监控
